ELYSIAN Trading System — Architecture

Updated 2026-03-30  |  5-Stage Quantitative Pipeline

1. Full Pipeline Overview StrategyRunner → Exchange → EventBus → RebalanceFSM

The Elysian pipeline runs five sequential stages per rebalance cycle. Data enters through exchange WebSocket feeds, propagates through the event bus to strategies, flows through weight computation, risk validation, and execution, and loops back via order-update events to keep the portfolio and shadow books in sync.

2. Key Data Contracts signals.py — frozen dataclasses

These immutable dataclasses form the typed contracts between pipeline stages. Each stage reads from the previous stage's output contract and enriches it.

Contract Producer Consumer Key fields
TargetWeights SpotStrategy.compute_weights() RebalanceFSM → PortfolioOptimizer weights: Dict[str,float] · timestamp · strategy_id · price_overrides
ValidatedWeights PortfolioOptimizer.validate() ExecutionEngine.execute() original: TargetWeights · weights (adjusted) · clipped · rejected · rejection_reason
OrderIntent ExecutionEngine.compute_order_intents() ExecutionEngine._submit_order() · Portfolio.lock_for_order() symbol · venue · side · quantity · order_type · price · strategy_id
RebalanceResult ExecutionEngine.execute() RebalanceFSM._on_enter_executing() · ShadowBook lock intents · submitted · failed · errors · submitted_orders (order_id → intent)

3. Portfolio & ShadowBook — Accounting Ledgers portfolio.py · shadow_book.py

Portfolio is the exchange-level source of truth (one per venue). ShadowBook is a per-strategy virtual ledger that tracks the capital slice attributable to each strategy. Both maintain identical internal state shapes and process OrderUpdateEvents identically.

4. Sub-Account Mode & Multi-Strategy Architecture WeightAggregator · ShadowBook

Multiple strategies share one exchange connector but maintain separate capital slices via ShadowBooks. The WeightAggregator merges target weights scaled by each strategy's allocation fraction. In sub-account mode each strategy may have a private event bus for routing only its own user-data events.

5. Component Summary source file reference

Component Source file Stage Responsibility
BinanceKlineFeed / BinanceOrderBookFeed connectors/BinanceDataConnectors.py 1 WebSocket kline & order-book feeds, managed by shared client managers. Emit KlineEvent / OrderBookUpdateEvent.
BinanceFuturesKlineFeed / BinanceFuturesOrderBookFeed connectors/BinanceFuturesDataConnectors.py 1 Futures-venue equivalents connecting to fapi.binance.com.
BinanceUserDataClientManager connectors/BinanceDataConnectors.py 1 New WS API (wss://ws-api.binance.com). HMAC-SHA256 via userDataStream.subscribe.signature. Emits OrderUpdateEvent + BalanceUpdateEvent. Reconnects with exponential backoff (1s→60s).
SpotStrategy / EventDrivenStrategy strategy/base_strategy.py 2 Base class. Subscribes to EventBus hooks (on_kline, on_orderbook_update, on_order_update). Calls request_rebalance() which triggers RebalanceFSM.
WeightAggregator core/weight_aggregator.py 2 Multi-strategy weight registry. Each strategy updates its weights; aggregator merges them scaled by allocation fraction. Thread-safe via asyncio.Lock.
RebalanceFSM core/rebalance_fsm.py 3 Drives IDLE→COMPUTING→VALIDATING→EXECUTING→COOLDOWN pipeline. Subclasses BaseFSM. Publishes RebalanceCycleEvent and RebalanceCompleteEvent.
PortfolioOptimizer risk/optimizer.py 4 6-stage stateless constraint projector: rate-limit → symbol filter → per-asset clip → exposure scale → cash floor → turnover cap. Returns ValidatedWeights.
ExecutionEngine execution/engine.py 5 Converts ValidatedWeights to OrderIntents. Sells first, then buys. Locks balance per submitted order. Returns RebalanceResult.
Portfolio core/portfolio.py ledger Shared-account position/cash ledger. Event-driven via _on_kline / _on_balance / _on_order_update. Maintains fill_tracker for incremental fills, locked_cash for LIMIT orders.
ShadowBook core/shadow_book.py ledger Per-strategy virtual ledger. Mirrors Portfolio state shape. Routes only its own strategy's orders via _order_strategy_map. Supports sub-account mode with private event bus.
BaseFSM / PeriodicTask core/fsm.py infra Async FSM base with re-entrancy guard and queued transition draining. PeriodicTask replaces bare while-True loops.
StrategyRunner run_strategy.py infra Top-level orchestrator. Wires all components, calls asyncio.gather(data feeds, strategy.run_forever()). Manages RunnerState: IDLE→RUNNING→STOPPING→STOPPED/FAILED.
