1. System Overview UPDATED Top-level component topology

Elysian now supports three asset-type pipelines: Spot, Perpetual Futures, and Isolated Margin. A dedicated MarketDataService container owns all WebSocket feed managers and publishes market data events to Redis pub/sub. Each strategy runs in its own container, subscribing to the Redis channels relevant to its symbols. Multi-sub-account strategies (mixing SPOT long-book with isolated MARGIN short-book) are supported via the new MarginStrategyRunner and MarginShadowBook.

Key Changes — Margin Trading Integration

  • IsolatedMarginExchangeConnector — new abstract base class extending SpotExchangeConnector with margin-specific collateral/borrowed/interest ledgers, borrow/repay/transfer_in/transfer_out operations, and periodic margin poll loop (every 60 s)
  • BinanceMarginExchange — concrete isolated margin connector; uses per-symbol listen keys via POST /sapi/v1/userDataStream/isolated; defaults to AUTO_BORROW_REPAY sideEffectType
  • BinanceMarginUserDataClientManager — one WebSocket reader task per symbol (vs. one global for spot); keepalive every 55 min
  • MarginShadowBook — extends ShadowBook; NAV formula subtracts borrowed + interest; overrides _adjust_cash(), _track_commission(), _on_balance_update() to route to _collateral dict instead of _cash
  • SubAccountKey + SubAccountPipeline — new routing primitives for multi-sub-account strategies
  • MarginStrategy — extends SpotStrategy; _pipelines dict keyed by SubAccountKey; per-pipeline FSMs; concurrent dispatch via asyncio.gather
  • MarginStrategyRunner — extends StrategyRunner; builds one SubAccountPipeline per sub_accounts YAML entry; injects MarginShadowBook for MARGIN type

4. Redis Transport Layer JSON serialisation codec

The transport layer is unchanged in design. The key point for margin is that channel_prefix(Venue.BINANCE, AssetType.MARGIN) returns "elysian:md:binance:spot" — identical to the spot prefix — because AssetType.MARGIN is remapped to AssetType.SPOT inside the function. This means margin strategies call strategy_channels() with AssetType.SPOT (or the runner does the remapping) so they subscribe to the correct channels.

5. Six-Stage Rebalance Pipeline Data flow through a single rebalance cycle

The six-stage pipeline is identical for SPOT and MARGIN strategies. The only difference is in Stage 5 (Execution): MARGIN OrderIntent carries asset_type=AssetType.MARGIN and side_effect=MarginSideEffect.AUTO_BORROW_REPAY, and the ExecutionEngine passes side_effect as an extra kwarg to place_market_order() / place_limit_order() — which BinanceMarginExchange picks up and sends as sideEffectType to the Binance API.

9. Position & ShadowBook UPDATED

ShadowBook is the standard ledger for SPOT and FUTURES strategies. MarginShadowBook extends it with isolated margin state. Both are registered in the Portfolio for NAV aggregation. Key additions are detailed in Section 16.

ShadowBook NAV Formulas

TypeNAV FormulaCash Field
ShadowBook (SPOT/FUTURES)_cash + Σ(position.qty × mark_price)_cash (USDT free balance)

Short positions in MarginShadowBook NAV Short positions (qty < 0) are intentionally excluded from the NAV sum. Their USDT proceeds from the sale are already captured in _collateral["USDT"]. Including negative position values would double-subtract the short exposure and understate NAV. Only long positions (qty > 0) contribute to the position term of the formula.

15. Isolated Margin Architecture MARGIN NEW

Isolated margin in Elysian maps each sub-account entry in the strategy YAML to a dedicated BinanceMarginExchange connector. Each isolated margin symbol has its own WebSocket listen key, its own collateral wallet, and its own position book tracked in MarginShadowBook. The Binance AUTO_BORROW_REPAY side-effect type means the system never needs to call borrow() / repay() manually — Binance handles the loan lifecycle automatically on each order.

Why periodic polling for margin accounts? Binance does NOT push interest accrual events via the user-data WebSocket stream. Interest is charged hourly (or at each loan repayment). The _margin_poll_loop() polls GET /sapi/v1/margin/isolated/account every 60 seconds to keep _accrued_interest current. Without this, the NAV calculation would overstate portfolio value between poll cycles by the amount of unpaid interest.
